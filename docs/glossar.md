# Glossar

Hier findest du die wichtigsten Fachbegriffe des Kurses mit kurzen Erklärungen. Begriffe, die im Kurs häufig vorkommen, werden auf allen Seiten automatisch mit einem Tooltip versehen. Klicke auf einen unterstrichenen Begriff, um die ausführliche Definition hier im Glossar zu sehen.

---

## A

<span id="api"></span>**API** – Application Programming Interface
:   Eine Schnittstelle, über die Softwarekomponenten miteinander kommunizieren. Im Kurs bauen wir REST-APIs mit FastAPI.

<span id="aws"></span>**AWS** – Amazon Web Services
:   Cloud-Computing-Plattform von Amazon. Wir nutzen EC2 (Server), S3 (Speicher) und weitere Dienste.

---

## B

<span id="bash"></span>**Bash**
:   Eine Unix-Shell und Kommandosprache. Standard-Shell auf den meisten Linux-Systemen.

<span id="bios"></span>**BIOS** – Basic Input/Output System
:   Firmware, die beim Start des Computers die Hardware initialisiert.

---

## C

<span id="ci/cd"></span>**CI/CD** – Continuous Integration / Continuous Deployment
:   Automatisierte Prozesse zum Testen und Ausliefern von Software.

<span id="cli"></span>**CLI** – Command-Line Interface
:   Textbasierte Schnittstelle zur Bedienung von Programmen über die Kommandozeile.

<span id="container"></span>**Container**
:   Eine leichtgewichtige, isolierte Umgebung zum Ausführen von Anwendungen. Docker ist die bekannteste Container-Plattform.

<span id="cors"></span>**CORS** – Cross-Origin Resource Sharing
:   Sicherheitsmechanismus im Browser, der regelt, welche Webseiten auf welche APIs zugreifen dürfen.

<span id="crud"></span>**CRUD** – Create, Read, Update, Delete
:   Die vier grundlegenden Operationen für die Arbeit mit Daten in einer Datenbank.

<span id="css"></span>**CSS** – Cascading Style Sheets
:   Sprache zur Gestaltung von Webseiten (Farben, Layout, Schriften).

<span id="csr"></span>**CSR** – Client-Side Rendering
:   Clientseitige HTML-Generierung. Der Browser rendert die Seite mit JavaScript.

<span id="cidr"></span>**CIDR** – Classless Inter-Domain Routing
:   Methode zur effizienten IP-Adressierung und Subnetzbildung.

<span id="cpu"></span>**CPU** – Central Processing Unit
:   Der Prozessor eines Computers, der Berechnungen und Befehle ausführt.

---

## D

<span id="devops"></span>**DevOps**
:   Kombination von Entwicklung (Development) und IT-Betrieb (Operations). Ziel: schnellere und zuverlässigere Software-Auslieferung.

<span id="dhcp"></span>**DHCP** – Dynamic Host Configuration Protocol
:   Protokoll zur automatischen Zuweisung von IP-Adressen in einem Netzwerk.

<span id="dns"></span>**DNS** – Domain Name System
:   System, das Domainnamen (z.B. google.com) in IP-Adressen übersetzt.

<span id="docker"></span>**Docker**
:   Plattform zur Containerisierung von Anwendungen. Ermöglicht einheitliche Ausführung unabhängig vom Betriebssystem.

<span id="dom"></span>**DOM** – Document Object Model
:   Baumstruktur-Darstellung eines HTML-Dokuments, die mit JavaScript manipuliert werden kann.

---

## E

<span id="ec2"></span>**EC2** – Elastic Compute Cloud
:   AWS-Dienst für virtuelle Server in der Cloud.

---

## F

<span id="fastapi"></span>**FastAPI**
:   Modernes Python-Web-Framework für den Bau von REST-APIs. Schnell, typsicher und mit automatischer Dokumentation.

<span id="fullstack"></span>**Fullstack**
:   Entwicklung, die sowohl Frontend (Benutzeroberfläche) als auch Backend (Server, Datenbank) umfasst.

---

## G

<span id="git"></span>**Git**
:   Verteiltes Versionskontrollsystem zur Nachverfolgung von Code-Änderungen.

<span id="gui"></span>**GUI** – Graphical User Interface
:   Grafische Benutzeroberfläche mit Fenstern, Buttons und Menüs.

---

## H

<span id="html"></span>**HTML** – Hypertext Markup Language
:   Auszeichnungssprache für die Struktur von Webseiten.

<span id="http"></span>**HTTP** – Hypertext Transfer Protocol
:   Protokoll für die Datenübertragung im Web. Basis aller Web-Kommunikation.

<span id="https"></span>**HTTPS** – HTTP Secure
:   Verschlüsselte Variante von HTTP für sichere Datenübertragung.

<span id="hdd"></span>**HDD** – Hard Disk Drive
:   Magnetische Festplatte zur Datenspeicherung.

---

## I

<span id="iam"></span>**IAM** – Identity and Access Management
:   AWS-Dienst zur Verwaltung von Benutzern, Gruppen und Zugriffsrechten.

<span id="ide"></span>**IDE** – Integrated Development Environment
:   Entwicklungsumgebung mit Editor, Debugger und weiteren Tools (z.B. VS Code, PyCharm).

<span id="ipv4"></span>**IPv4** – Internet Protocol Version 4
:   Netzwerkprotokoll mit 32-Bit-Adressen (z.B. 192.168.1.1).

<span id="ipv6"></span>**IPv6** – Internet Protocol Version 6
:   Netzwerkprotokoll mit 128-Bit-Adressen für den wachsenden Bedarf an IP-Adressen.

---

## J

<span id="json"></span>**JSON** – JavaScript Object Notation
:   Leichtgewichtiges Datenaustauschformat, lesbar für Menschen und Maschinen.

<span id="jsx"></span>**JSX** – JavaScript XML
:   Syntaxerweiterung für React, die HTML-ähnlichen Code in JavaScript ermöglicht.

<span id="jwt"></span>**JWT** – JSON Web Token
:   Standard für tokenbasierte Authentifizierung in Web-Anwendungen.

---

## L

<span id="linux"></span>**Linux**
:   Open-Source-Betriebssystem. Basis vieler Server und Cloud-Dienste.

<span id="lpi"></span>**LPI** – Linux Professional Institute
:   Organisation, die Linux-Zertifizierungen anbietet (z.B. Linux Essentials).

---

## M

<span id="mongodb"></span>**MongoDB**
:   Dokumentenorientierte NoSQL-Datenbank, speichert Daten als JSON-ähnliche Dokumente.

---

## N

<span id="nosql"></span>**NoSQL** – Not Only SQL
:   Sammelbegriff für Datenbanken, die nicht auf dem relationalen Modell basieren (z.B. MongoDB, Redis).

<span id="npm"></span>**npm** – Node Package Manager
:   Paketmanager für JavaScript. Verwaltet Abhängigkeiten in Node.js-Projekten.

---

## O

<span id="oop"></span>**OOP** – Objektorientierte Programmierung
:   Programmierparadigma, das Daten und Funktionen in Klassen und Objekten organisiert.

<span id="orm"></span>**ORM** – Object-Relational Mapping
:   Technik, die Objekte im Code auf Datenbanktabellen abbildet (z.B. SQLAlchemy).

---

## P

<span id="postgresql"></span>**PostgreSQL**
:   Leistungsfähiges, relationales Open-Source-Datenbanksystem.

<span id="powershell"></span>**PowerShell**
:   Kommandozeilen-Shell und Skriptsprache von Microsoft für Windows-Administration.

<span id="python"></span>**Python**
:   Vielseitige Programmiersprache, bekannt für einfache Syntax. Wird im Kurs für Backend-Entwicklung verwendet.

---

## R

<span id="ram"></span>**RAM** – Random Access Memory
:   Arbeitsspeicher eines Computers für schnellen Datenzugriff.

<span id="react"></span>**React**
:   JavaScript-Bibliothek von Meta für den Bau von Benutzeroberflächen mit Komponenten.

<span id="redis"></span>**Redis**
:   In-Memory-Datenbank, häufig als Cache eingesetzt für schnelle Datenzugriffe.

<span id="rest"></span>**REST** – Representational State Transfer
:   Architekturstil für Web-APIs. Nutzt HTTP-Methoden (GET, POST, PUT, DELETE).

---

## S

<span id="s3"></span>**S3** – Simple Storage Service
:   AWS-Dienst zur Speicherung von Dateien in der Cloud.

<span id="smtp"></span>**SMTP** – Simple Mail Transfer Protocol
:   Protokoll zum Versenden von E-Mails.

<span id="spa"></span>**SPA** – Single Page Application
:   Webanwendung, die auf einer einzigen HTML-Seite läuft und Inhalte dynamisch nachlädt.

<span id="sql"></span>**SQL** – Structured Query Language
:   Standardsprache für die Abfrage und Manipulation relationaler Datenbanken.

<span id="ssh"></span>**SSH** – Secure Shell
:   Verschlüsseltes Protokoll für sichere Verbindungen zu entfernten Servern.

<span id="ssr"></span>**SSR** – Server-Side Rendering
:   Serverseitige HTML-Generierung. Der Server liefert fertig gerendertes HTML.

<span id="ssd"></span>**SSD** – Solid State Drive
:   Flash-Speicher-Festplatte, schneller als herkömmliche HDDs.

---

## T

<span id="tcp"></span>**TCP** – Transmission Control Protocol
:   Verbindungsorientiertes Netzwerkprotokoll für zuverlässige Datenübertragung.

---

## U

<span id="udp"></span>**UDP** – User Datagram Protocol
:   Verbindungsloses Netzwerkprotokoll für schnelle, aber unzuverlässige Datenübertragung.

<span id="uefi"></span>**UEFI** – Unified Extensible Firmware Interface
:   Moderner Nachfolger des BIOS für die Hardware-Initialisierung.

<span id="url"></span>**URL** – Uniform Resource Locator
:   Webadresse, die den Standort einer Ressource im Internet angibt.

<span id="uri"></span>**URI** – Uniform Resource Identifier
:   Eindeutige Kennung für eine Ressource, allgemeiner als eine URL.

---

## V

<span id="venv"></span>**VENV** – Virtual Environment
:   Isolierte Python-Umgebung, um Projektabhängigkeiten getrennt zu halten.

<span id="vm"></span>**VM** – Virtuelle Maschine
:   Software-Emulation eines Computers. Ermöglicht die Ausführung eines Betriebssystems innerhalb eines anderen.

<span id="vpc"></span>**VPC** – Virtual Private Cloud
:   Isoliertes virtuelles Netzwerk innerhalb einer Cloud-Plattform (z.B. AWS).

---

## Y

<span id="yaml"></span>**YAML** – YAML Ain't Markup Language
:   Menschenlesbares Format für Konfigurationsdateien (z.B. docker-compose.yaml, mkdocs.yml).
