# Modulabschlussprojekt – Frontend & Fullstack

In den letzten drei Wochen von Modul 3 (Woche 10 bis 12) entwickelt ihr in **Teamarbeit** eine vollständige Multitier-Webanwendung. Das Projekt verbindet alles, was ihr im Modul gelernt habt: React-Frontend, FastAPI-Backend, Datenbank und Cloud-Deployment.

---

## Anforderungen

| Bereich | Anforderung |
|---------|-------------|
| **Frontend** | React mit JavaScript |
| **Backend** | FastAPI mit Python |
| **Datenbank** | Mindestens eine relationale Datenbank (PostgreSQL empfohlen) |
| **Deployment** | Anwendung auf einer EC2-Instanz erreichbar (Mindestziel) |
| **Versionskontrolle** | Gemeinsames GitHub-Repository |
| **Teamarbeit** | Scrum-ähnliche Arbeitsweise mit Daily Standups |

---

## Bewertung

| Anteil | Was wird bewertet |
|--------|-------------------|
| **30%** | Code (Funktionalität, Struktur, Qualität) |
| **70%** | Präsentation der Anwendung am **27.03.** vor Publikum |

## Ablauf & Arbeitsweise

### Tagesablauf

1. **09:00 Uhr** – Check-in im [Klassenzimmer](https://meet.google.com/qnu-awsy-euv) (Anwesenheit)
2. Danach arbeiten die Teams eigenständig
3. Jedes Team hält ein **Daily Standup** ab

### Daily Standup (Scrum)

Im Daily bespricht jedes Teammitglied:

- **Was habe ich gestern gemacht?**
- **Was mache ich heute?**
- **Gibt es Hindernisse?**

Die Daily-Ergebnisse werden dokumentiert und von **allen Teammitgliedern** eingereicht.

### Sprints & Terminplan

Wir arbeiten in Sprints, ähnlich wie im letzten Abschlussprojekt. Der genaue **Terminplan mit allen wichtigen Terminen** wird am Montag veröffentlicht.
